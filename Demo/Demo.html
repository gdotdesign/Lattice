Effects, transitions, shadows

Every effect, transition, or shadow you see is done with CSS.

This is because the UI is CSS driven. That means that all the UI elements position, styles are defined in the css and the library only modifies what is needed to achive the functionality of the element.

To make it simple, if you want to make an element dissapear you only have to do: element.setStyle('opacity',0); and add a css property so it would transition, this way the UI don't constrain the developer, more like empower them to make stunning effects with CSS.

With that in mind all the elements can have different css classes by theme, so it won't conflict with your webpage / webapp.

Every element can be "grabbed"

This UI is based heavily on Mootools class architetcure. Most elements are extended upon a base class (Core.Abstract, Data.Abstract), and these classes implement the toElement method, so every UI element can be grabbed, adopted, disposed, etc.

Why webkit?

Keep in mind that this project is for offline web applications in the upcoming Google Web Store. Because of this there is no logical reason to make it work with non webkit browsers, also G.UI implements a number of technologies that are not yet available for other browsers: presistent local storage, DOMNodeInsertedIntoDocument event, transitions, css effects just to name a few. These technologies make the project work, without these it is just another UI library.

There are plans to make it cross browser. It just needs some clever workarounds. If you are interested to make this happen and have some ideas contact me trought github.

Pickers

A picker is a floating box containing data which you can choose from. It usually appears when you click on an input element.
Pickers in G.UI are pretty much the same with a few differences:

  • It will appear on the side of the element where is more space. (it will never go out of the window)
  • You can create custom data pickers.
  • You can attach it to any html element.
  • Automatically highlight the element which initiated the "pick" by adding a class to it specified by the theme.
  • You can change the trigger event (click, double click, etc.).

Usage

Basic usage of the built in pickers: Pickers.Number.attach($('numberSelect')); If you have a custom data type (Data.MyDatatype) then: Pickers.MyDatatype = new Picker({type:"MyDatatype"});
Pickers.MyDatatype.attach($('inputbox')); When the data of the picker changes (when you pick a color, change text, change date, etc.) the attached element will fire the 'change' event with the value of the picked data as the argument, also if it is an input element it will change it's value. Pickers.Number.attach($('numberSelect'));
$('numberSelect').addEvent('change',function(value){
    console.log('The current value of the picker is: '+value);
})

Theme properties

  • class - css class of the base element.
  • event - trigger event
  • picking - css class of the highlight
  • offset - offset from the trigger element
GDotUI.Theme.Picker:{
    'class':'picker',
    event: 'click',
    picking: 'picking',
    offset: 10
}
